# Instagram DM 자동화 매크로 — 요구사항 정의서

> **출처**: `ref-inpk-screenshots/` 내 15장의 경쟁사(인포크링크 추정) 스크린샷 reverse engineering 결과
> **목적**: 동일한 사용자 경험·기능 범위로 DM 자동화 매크로를 재구현하기 위한 기능 명세
> **범위**: 모바일(iOS) UI 기반. 화면별 UI 컴포넌트, 인터랙션, 분기 로직, 데이터 모델 정의
> **유의**: 본 문서는 스크린샷에서 직접 관찰 가능한 정보만을 기반으로 작성했으며, 추측이 들어간 부분은 별도 표시(⚠︎)함

---

## 1. 기능 개요

크리에이터(인플루언서/판매자)가 Instagram 게시물에 달린 댓글을 트리거로, 자동으로 DM을 발송하는 매크로 기능. 댓글 키워드 매칭, 팔로우 여부 분기, 자동 답글, 미리 정의된 템플릿 사용 등을 지원.

### 1.1 핵심 가치 제안 (스크린샷 카피 기반)
- "24시간 언제나 스마트하게 응대하세요" (scene_004)
- 템플릿으로 간편하게 등록 (scene_018)

### 1.2 진입점

| # | 경로 | 화면 |
|---|---|---|
| A | 홈 → 크리에이터 도구 카드 → **DM 자동화하기** | scene_004 |
| B | 하단 탭 **매니저** → **DM 자동화** 토글 | scene_018, scene_047 |

## 2. 전체 사용자 플로우

```
[홈/매니저 진입]
    │
    ├──▶ [매니저 화면: DM 자동화 탭]  ◀── 기본 hub
    │       ├─ 템플릿 선택 (가로 스크롤)
    │       ├─ + 자동화 추가하기 (직접 생성)
    │       └─ 기존 자동화 목록 (게시물 카드)
    │
    ▼
[템플릿 적용 알림]  (scene_006)
"선택하신 템플릿을 불러왔어요 / 버튼을 눌러 설정을 확인해주세요"
    │
    ▼
[게시물 선택 모드 선택]
    ├─ 자동화하려는 게시물을 직접 선택  ──▶ [IG 게시물 그리드] (scene_007)
    └─ 다음에 생성되는 게시물을 자동화
    │
    ▼
[자동화 설정 메인]  (scene_019) — 4개 섹션 아코디언
    ├─ ① 답글을 남겨볼까요?
    ├─ ② 팔로우 여부에 따라 다르게 보낼까요?
    ├─ ③ 어떤 댓글에 메시지를 보낼까요?
    └─ ④ 어떤 게시물을 자동화할까요?
    │
    ▼ [다음]
[자동화할 메시지 설정]  (scene_046) — 텍스트/이미지 캐러셀
    │
    ▼
[버튼 설정 — 옵션]  (scene_012, scene_014)
    │
    ▼
[고객에게 전송될 메시지 확인]  (scene_008, scene_017)
    │
    ▼ [완료]
[매니저 화면 복귀, 자동화 카드 추가됨]  (scene_047)
```

## 3. 화면별 상세 명세

### 3.1 홈 진입점 (scene_004)

| 요소 | 내용 |
|---|---|
| 섹션 타이틀 | "크리에이터 도구" |
| 카드 | "DM 자동화하기" / 부제 "24시간 언제나 스마트하게 응대하세요" / IG 아이콘 |
| 동반 카드 | "링크 관리하기", "내 링크 분석 보기" |
| 인터랙션 | 카드 탭 → 매니저(DM 자동화) 진입 ⚠︎(추정) |

### 3.2 매니저 — DM 자동화 hub (scene_018, scene_047)

#### 3.2.1 상단 영역
- 페이지 타이틀: "매니저"
- 사용자 핸들 표시: `wonseok.choi.3 v` (계정 전환/선택용 dropdown ⚠︎)
- 안내 배너: "매니저 이용이 처음인가요? / 서비스 이용 가이드 바로가기"

#### 3.2.2 탭
- **DM 자동화** (활성)
- **이벤트 추첨**

#### 3.2.3 템플릿 영역 — "템플릿으로 간편하게 등록해 보세요"
가로 스크롤 카드 리스트. 첫 번째는 **`+`** (빈 자동화 신규 생성):

| 템플릿 ID | 카테고리 | 이름 |
|---|---|---|
| `+` | (직접 생성) | — |
| TEMPLATE_COMMENT_EVENT | 판매 자동화 | 댓글 이벤트 템플릿 |
| TEMPLATE_PRODUCT_INFO | 판매 자동화 | 제품 정보 전달 템플릿 |
| TEMPLATE_PURCHASE_AUTH | 판매 자동화 | 구매 인증 템플릿 |

#### 3.2.4 자동화 카운트 & 액션
- 좌측: `총 N개` (scene_018: 4개, scene_047: 5개 — 자동화 추가 시 증가 확인)
- 우측: **`+ 자동화 추가하기`** primary 버튼 (orange, FAB 스타일)

#### 3.2.5 정렬
- "게시물 순" / "최신 순" — 토글성 정렬 옵션 ⚠︎

#### 3.2.6 자동화 게시물 카드 (반복 리스트)
| 요소 | 설명 |
|---|---|
| 게시물 썸네일 | IG 게시물 이미지 |
| 게시물 제목 | "설정된 타이틀 없음" (기본값) |
| 키워드 chip | 예: `즉시 1`, `참여` (트리거 키워드/모드 표시) |
| 상태 라벨 | "즉시 발송" |
| 활성 토글 | on/off (예: 토글 색상 활성=blue) |
| `...` 메뉴 | 추가 액션(편집/삭제 등) ⚠︎ |
| 메시지 미리보기 | 자동 발송될 DM 본문 1~2줄 |
| 통계 | `전송 0` / `읽음 0` / `클릭 0` |

### 3.3 템플릿 적용 알림 (scene_006)

전체 화면 dim + 중앙 체크 아이콘:
- 카피: "선택하신 템플릿을 불러왔어요 / 버튼을 눌러 설정을 확인해주세요"
- 하단 sheet: "**인스타그램 게시물을 선택해주세요**"
  - ◉ 자동화하려는 게시물을 직접 선택 할게요 (선택)
  - ○ 다음에 생성되는 게시물을 자동화 할게요

### 3.4 게시물 그리드 선택 (scene_007)

- 타이틀 동일, sheet 확장 상태
- IG 게시물 thumbnail grid (3 column)
- 단일 선택 ⚠︎(다중 선택 가능 여부는 스크린샷에서 미확인)
- 하단 **`다음`** 버튼 (선택 전: disabled / 선택 후: enabled로 추정)

### 3.5 자동화 설정 메인 (scene_019)

타이틀: "**인스타그램 메시지 자동화 설정을 선택해주세요**"

아코디언 4개 섹션. 각 섹션은 우측에 `(완료)` 라벨 + chevron.

| # | 섹션 질문 | 현재 값 (예시) | 펼친 화면 |
|---|---|---|---|
| ① | 답글을 남겨볼까요? | 총 4개 답글 설정 | scene_025, scene_026, scene_028 |
| ② | 팔로우 여부에 따라 다르게 보낼까요? | 동일하게 전송 | (펼친 화면 미수집) |
| ③ | 어떤 댓글에 메시지를 보낼까요? | 참여 | scene_022 |
| ④ | 어떤 게시물을 자동화할까요? | 직접 선택 | (scene_006~007 흐름) |

하단: **`다음`** primary 버튼

#### 3.5.1 [섹션 ①] 답글 설정 (scene_025, scene_026, scene_028)

- 타이틀 옆 카운트: "총 4개 답글 설정" (개수 동적)
- 옵션 (single select):
  - ○ 아니요, 남기지 않을래요
  - ◉ 네, 랜덤하게 답글을 남겨볼래요
- 선택 시 톤(어조) 탭 노출:
  - **정중하게** (활성, orange)
  - 친근하게
  - 직접 ← 직접 작성 모드
- 답글 리스트 (각 항목 우측 `−` 삭제 버튼):
  - 예시(scene_025): "이벤트 참여 완료되었어요!", "소중한 댓글 감사합니다 🚀", "이벤트에 참여해주셔서 감사해요 🥹", "참여 완료! 확인 되셨어요? ❤️"
  - 예시(scene_028, 5개): "링크 전송해 드렸습니다", "소중한 댓글 감사드립니다! 링크 보내드렸습니다", "댓글 감사합니다! 메시지함 확인 부탁드립니다", "메시지 확인 부탁드립니다! 감사합니다", "링크 전송 완료했습니다! 요청함도 확인해 주세요"
- **`+ 새로운 답글 추가하기`** (점선 박스 버튼)
- 답글 추가 시 새 입력 필드가 리스트 하단에 추가되며 키보드 활성화 (scene_026)

#### 3.5.2 [섹션 ②] 팔로우 분기 ⚠︎
- 현재 값: "동일하게 전송"
- 추정 옵션: 동일하게 전송 / 팔로워에게만 / 비팔로워에게만 / 분기별 다른 메시지 (스크린샷 미수집)

#### 3.5.3 [섹션 ③] 댓글 트리거 (scene_022)
- 현재 값 표기: "참여, 좌표"
- 옵션 (single select):
  - ○ 댓글을 달기만 하면 모두 전송할래요
  - ◉ 특정 키워드를 남기면 전송할래요
- 키워드 입력 필드: placeholder "발송하시는 키워드를 입력해주세요"
- 헬프 텍스트: "플러스(+)나 엔터를 눌러서 추가할 수 있어요"
- 입력된 키워드 chip: `참여 ×`, `좌표 ×` (개별 삭제 가능)

#### 3.5.4 [섹션 ④] 대상 게시물 (scene_019의 "직접 선택")
- 현재 값: "직접 선택"
- 펼친 화면은 scene_006/scene_007의 흐름 재사용 ⚠︎

### 3.6 메시지 설정 (scene_046)

타이틀: "**자동화할 메시지를 설정해주세요**"

| 요소 | 설명 |
|---|---|
| 모드 탭 | **텍스트** / **이미지 캐러셀** (scene_046은 이미지 캐러셀 활성) |
| 미리보기 | 채팅 버블 형태. 본문 + 버튼 영역(`아이콘엑스`) + `+`(버튼 추가) |
| 메시지 input | 필수 (`*`). 본문 텍스트 입력 |
| 하단 CTA | **`다음`** primary |

### 3.7 버튼 설정 (scene_012, scene_014)

진입: scene_046의 미리보기 내 `+`(버튼 추가) 또는 기존 버튼 탭 → 하단 sheet 노출

| 요소 | 설명 |
|---|---|
| 헤더 | "**버튼 설정**" + 우측 **`링크 블록 불러오기 ↻`** 보조 액션 |
| 버튼 텍스트 (필수) | text input. 예: "아이콘엑스" |
| 클릭 시 동작 (필수) | 좌측 dropdown: **URL** (다른 옵션 존재 가능 ⚠︎) + 우측 input |
| URL input placeholder | "전결하고 싶은 링크 주소전체를" ⚠︎(OCR 추정) |
| URL input 예시 | `iconx.co.kr` |
| CTA | **`설정 완료`** primary |

### 3.8 메시지 확인 (scene_008, scene_017)

타이틀: "**고객에게 전송될 메시지를 확인해주세요**"

- 채팅 미리보기 (회색 아바타 + 버블)
  - 본문 + 버튼(예: "인포크링크 바로가기") + `+`(버튼 추가)
  - tooltip 노출 가능: "버튼 설정 위치가 변경되었어요" (scene_008)
- 하단 메시지 텍스트 영역 (필수, `*`)
- 하단 CTA: **`완료`** primary

> scene_008(상세 미리보기 + 버튼 포함) vs scene_017(단순 메시지 + 버튼 미설정) — **버튼 유무에 따라 같은 화면이 다르게 렌더링됨**

### 3.9 자동화 추가 후 hub 복귀 (scene_047)

- 총 N개 카운터 +1
- 신규 자동화 카드가 리스트 상단(또는 정렬 기준에 따라)에 노출
- 활성 토글 기본 ON, 메시지 미리보기/키워드 chip 표시

## 4. 데이터 모델 (제안)

스크린샷 관찰을 바탕으로 도출한 핵심 엔티티:

```
Automation {
  id: string
  user_id: string                       // wonseok.choi.3 등
  template_id: enum?                    // COMMENT_EVENT | PRODUCT_INFO | PURCHASE_AUTH | null
  status: enum                          // ACTIVE | INACTIVE
  delivery_mode: enum                   // IMMEDIATE ("즉시 발송")
  title: string?                        // 기본 "설정된 타이틀 없음"

  // ④ 대상 게시물
  target_mode: enum                     // SPECIFIC_POSTS | FUTURE_POSTS
  target_post_ids: string[]             // IG media IDs

  // ③ 댓글 트리거
  trigger_mode: enum                    // ALL_COMMENTS | KEYWORD_MATCH
  trigger_keywords: string[]            // ["참여", "좌표"]

  // ② 팔로우 분기
  follower_branch: enum                 // SAME | FOLLOWERS_ONLY | NON_FOLLOWERS_ONLY | DIFFERENT_MESSAGES ⚠︎

  // 메시지 본문
  message: {
    type: enum                          // TEXT | IMAGE_CAROUSEL
    body: string                        // 필수
    images: ImageRef[]?                 // 캐러셀 모드일 때
    buttons: Button[]                   // 0..n
  }

  // ① 답글
  reply: {
    enabled: boolean                    // false = "아니요, 남기지 않을래요"
    mode: enum                          // RANDOM (현재 관찰된 유일 옵션)
    tone: enum                          // POLITE | FRIENDLY | CUSTOM
    items: string[]                     // 답글 후보 1..n
  }

  // 통계
  stats: { sent: int, read: int, clicked: int }

  created_at, updated_at
}

Button {
  text: string                          // "아이콘엑스"
  action_type: enum                     // URL (관찰), 추가 옵션 가능 ⚠︎
  url: string?                          // "iconx.co.kr"
}
```

## 5. 분기·검증 규칙

### 5.1 필수 필드 (스크린샷에서 `*`로 표기)
- 메시지 본문
- 버튼 텍스트, 클릭 시 동작 (버튼을 추가한 경우)

### 5.2 분기 로직
- **target_mode = SPECIFIC_POSTS** → 게시물 그리드 선택 화면 노출 (scene_007)
- **target_mode = FUTURE_POSTS** → 게시물 선택 스킵, 향후 게시물에 자동 적용
- **trigger_mode = ALL_COMMENTS** → 키워드 입력 영역 hide
- **trigger_mode = KEYWORD_MATCH** → 키워드 chip 입력 활성, ≥1개 필요 ⚠︎
- **reply.enabled = false** → 톤 탭 / 답글 리스트 hide
- **message.type = TEXT** → 이미지 캐러셀 영역 hide / 반대도 동일
- 버튼 미추가 시 미리보기에서 버튼 영역 비표시 (scene_017)

### 5.3 상태 전이
- `자동화 추가하기` 또는 `템플릿 선택` → 신규 Automation draft 생성
- 모든 섹션 `(완료)` → `다음` 활성화 ⚠︎(필수 섹션 기준 추정)
- `완료` → status=ACTIVE, hub로 복귀
- 카드 토글 → ACTIVE ↔ INACTIVE

| 답글 삭제 | 항목 우측 `−` (마이너스 아이콘, 옅은 핑크 배경) |
| Chip 삭제 | chip 우측 `×` |
| Sheet | 하단에서 슬라이드업 (scene_006, scene_012) |
| 키보드 | iOS 한글/영문 키보드, 화면 상단 navigation `^ v ✓` |

## 8. 미해결/추가 확인 필요 항목 (Open Questions)

1. **섹션 ②(팔로우 여부)** 펼친 화면 미수집 → 옵션 종류 확정 필요
2. 게시물 그리드의 **다중 선택 가능 여부** 미확인
3. **이벤트 추첨** 탭(scene_018)의 별도 기능 명세 필요
4. `클릭 시 동작` dropdown의 URL 외 추가 옵션 (예: 메시지 발송, 인앱 페이지 이동 등) 미수집
5. `...` 메뉴(자동화 카드)의 액션 (편집/복제/삭제) 미수집
6. "랜덤하게 답글을 남겨볼래요" 외에 답글 발송 정책(순차 등) 존재 여부
7. 버튼 다중 추가 시 최대 개수 / 정렬 / 삭제 UX
8. 템플릿 적용 시 **사전 채워지는 필드** 범위 (메시지/답글/키워드 모두 vs 일부)
9. "다음에 생성되는 게시물을 자동화" 모드의 종료 조건 (수동 off vs 기간/횟수)
10. 통계(전송/읽음/클릭)의 집계 주기 및 클릭 추적 방식
