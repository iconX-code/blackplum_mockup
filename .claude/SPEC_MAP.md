# spec.md 섹션 라인 인덱스

> 본 문서는 spec.md를 부분 Read하기 위한 라인 범위 지도. spec.md 통째 Read는 컨텍스트 폭발(60k+) 사고 패턴이므로, sub-agent는 본 표에서 필요 섹션의 라인 범위 확인 후 Read tool의 `offset` / `limit`으로 정확히 점프할 것.
>
> 갱신 정책: spec.md에 새 §N 또는 §N-M이 추가/이동/삭제되면 본 파일도 함께 갱신. 같은 섹션 안의 본문 수정(±수십 줄)은 누적되다 다음 큰 변경 때 일괄 갱신해도 무방. 라인 수가 ±50줄 이상 어긋나기 시작하면 갱신 필요.

## 알려진 §0 / §10 특이사항
- §0(섹션 인덱스) 자체는 본 SPEC_MAP.md 도입 후 사실상 redundant이지만, spec.md 내부 self-doc으로 보존. 본 SPEC_MAP을 우선 참조.
